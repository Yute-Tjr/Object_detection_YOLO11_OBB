# YOLO11 OBB 高 IoU 优化实验

本文归档 YOLO11l OBB 基线形成过程。当前最终结果和运行命令见根目录 [README](../../README.md)。

## 1. 实验目标

模型在 mAP50 下已接近饱和，优化重点从是否检出转为旋转框边界和角度是否足够准确。核心观察指标为 `mAP80`、`mAP85`、`mAP90` 和 `mAP95`。

## 2. 关键实验结果

| 实验 | mAP80 | mAP85 | mAP90 | mAP95 |
| --- | ---: | ---: | ---: | ---: |
| 完整数据，batch4，degrees0 | 0.917490 | 0.805687 | 0.618882 | 0.195674 |
| 完整数据，batch8，degrees5 | 0.919840 | 0.821532 | 0.621163 | 0.171728 |
| 时间筛选，batch8，degrees0 | 0.930040 | 0.841242 | 0.700215 | 0.216485 |
| 时间筛选 + 去 index=-1，thin/thick | 0.962807 | 0.894190 | 0.711124 | 0.183112 |
| 时间筛选 + 去 index=-1，单一 label1 | 0.978489 | 0.896184 | 0.736873 | 0.220707 |

最后一行使用 6 类定义，其他 thin/thick 实验使用 7 类定义，因此整体 `all` 指标不能单独用于判断 label1 拆分是否更好。

## 3. label1 类别定义对照

在同一批 198/53 张图上，把 `label1_thin` 和 `label1_thick` 按 test 数量加权后，与单一 `label1` 比较：

| 类别方案 | label1 mAP80 | label1 mAP85 | label1 mAP90 | label1 mAP95 |
| --- | ---: | ---: | ---: | ---: |
| thin/thick 加权合并 | 0.918529 | 0.691935 | 0.440240 | 0.009434 |
| 单一 label1 | 0.937068 | 0.579722 | 0.349609 | 0.000405 |

单一 label1 的 mAP80 更高，但 thin/thick 在 mAP85、mAP90 和 mAP95 更好。逐图 IoU85 对齐中，thin/thick 通过而单一 label1 未通过的有 8 张，反向只有 1 张。因此当前主数据仍采用 thin/thick 分类。

## 4. 旋转增强实验

`degrees=5` 对 5-10 度倾斜样本以及部分 `label3/label5` 有帮助，但产生了三个问题：

- `label1_thin` 的高 IoU 定位明显下降。
- 固定置信度可视化中的 IoU85 失败行由 146 增至 162。
- 整体 mAP95 从 0.195674 降至 0.171728。

因此旋转增强没有作为当前默认配置。后续若继续试验，应使用小角度、类别稳定性和多 seed 共同判断，不能只看单次 all mAP90。

## 5. 数据清理实验

时间筛选使整体 mAP90 从 0.618882 提升至 0.700215，是早期最明显的增益。继续去除语义异常的 `index=-1` 样本后：

- mAP80：0.930040 -> 0.962807
- mAP85：0.841242 -> 0.894190
- mAP90：0.700215 -> 0.711124
- mAP95：0.216485 -> 0.183112

这说明清理能显著改善 mAP80/mAP85，并小幅提高 mAP90，但 mAP95 仍容易受细微边界差异影响。

## 6. 当前 YOLO11l 基线

最终保留配置：

```yaml
model: yolo11l-obb.pt
epochs: 50
imgsz: 1280
batch: 8
degrees: 0.0
seed: 42
data: datasets/obb_thin_thick/data.yaml
```

最终统一评估：

| precision | recall | mAP50 | mAP80 | mAP85 | mAP90 | mAP95 |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 0.997159 | 0.999863 | 0.995000 | 0.962807 | 0.894190 | 0.711124 | 0.183112 |

## 7. 阶段结论

- 当前瓶颈不是漏检，而是框角度和边界在严格 IoU 下的偏差。
- 提高输入分辨率、调整 batch 或增加普通增强没有表现出比数据清理更稳定的收益。
- `label1_thin`、`label1_thick`、`label3` 和 `label5` 是高 IoU 优化需要重点观察的类别。
- 所有后续模型都应在相同 test split、相同预测置信度和相同 Ultralytics OBB 指标口径下比较。
